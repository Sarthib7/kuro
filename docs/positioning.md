# Kuro Positioning

Updated: 2026-05-17

## Core Position

Kuro = open, self-hosted trading copilot for Solana operators who want AI help without handing keys to black-box apps.

Not "Minara clone."
Not "guaranteed alpha bot."
Not "consumer finance app."

## One-Liner

Open-source Solana trading copilot with local keys, dry-run first execution, and hard risk caps in Rust.

## Sharp Version

Kuro turns token, wallet, and market intent into source-backed research, dry-run trade previews, scoped workflows, and guarded Autopilot strategies. Your keys stay local. Executor enforces caps. LLM cannot bypass risk.

## ICP

Primary user:

- Solana power trader.
- Sniper / wallet watcher / memecoin operator.
- Comfortable funding hot wallet with limited size.
- Already uses DexScreener, GMGN, Jupiter, Photon/Axiom, Telegram alerts.
- Wants speed + automation, but distrusts closed bots.

Secondary user:

- Crypto builder/operator.
- Wants self-hosted agent infra.
- Wants auditable executor + API surface.
- Will tolerate CLI early.

Avoid initial ICP:

- New retail investor.
- Passive "make money for me" user.
- User needing financial advice.
- User wanting CEX-only bot.

## Category

Use:

- "Open trading copilot"
- "Self-hosted Solana execution agent"
- "Guarded Autopilot for Solana"

Avoid:

- "AI CFO" as main category.
- "Autonomous money printer."
- "Financial advisor."

## Main Wedge

Closed AI trading apps ask for trust.

Kuro gives proof:

- Code visible.
- Keys local.
- Live trades scoped.
- Dry-run default.
- Executor caps enforced outside prompt.
- Every action auditable.

## Messaging Pillars

### 1. Local Keys

User owns wallet. Kuro signs locally. No controller wallet pitch.

Copy:

> Trade through AI without giving AI custody.

### 2. Dry-Run First

Every trade path starts as preview/sim. Live needs explicit user action or active Trading Scope.

Copy:

> Ask for trade. Get full preview. Go live only when scope allows.

### 3. Executor-Enforced Risk

Rust Executor enforces per-trade, daily, drawdown, slippage, and venue caps.

Copy:

> Prompt can suggest. Executor decides what is allowed.

### 4. Solana-Native Speed

Kuro starts where speed matters: fresh pools, Jupiter, Jito, wallet streams.

Copy:

> Built for Solana hot paths, not generic portfolio chat.

### 5. Source-Backed Research

Reports cite token, wallet, market, and transaction data.

Copy:

> No mystery signals. Show data behind decision.

## Product Modes

### Copilot

User asks. Kuro researches, explains, previews.

Examples:

- "Analyze this mint."
- "Is this dev wallet risky?"
- "Preview 0.05 SOL buy if safe."

### Workflow

User defines trigger/condition/action graph.

Examples:

- "Notify if this wallet buys."
- "Dry-run buy if liquidity > X and top holders < Y."

### Autopilot

User activates Strategy inside Trading Scope.

Examples:

- Pump.fun safety sniper.
- Smart-money follower.
- DCA ladder.
- TP/SL rebalancer.
