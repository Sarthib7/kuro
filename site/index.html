Onboarding

Start with visibility, then unlock execution.

01

Run locally

Start the executor, install the agent, and use read-only token and Phoenix market commands.

npm run dev -- phoenix-markets SOL
02

Dry-run automation

Let the watcher and policy produce decisions without submitting transactions.

KURO_AUTONOMOUS_LIVE=false
03

Host privately

Deploy executor and agent as separate services, with the executor reachable only over private networking.

KURO_EXECUTOR_URL=http://kuro-executor.internal:7777
04

Flip live deliberately

Fund a capped hot wallet, pin live flags, and keep dry-run evidence before enabling any live path.

KURO_MAX_TRADE_SOL=0.02

Operating model

Two services, one authority boundary.

Executor

Rust service that owns the hot wallet, signs transactions, simulates Phoenix orders, and rejects requests that exceed caps.

Agent

Node worker that watches markets, runs analysis, calls LLM tools, and forwards allowed intents to the executor.

State

Volumes keep wallet, risk state, and positions stable across deploys while the next persistence layer is built.
